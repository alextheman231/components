## Description of Changes

<!-- user-editable-section-start -->
- Make createObjectQueryBoundary extend the item boundary rather than the base
  - It is actually still helpful to have access to `QueryBoundary.Data`, as there have been cases in Lexicon where mixing between the two is the most ideal approach.
  - `QueryBoundary.Data` works just fine with object-typed data, so it makes sense to give it access to the object boundary too.
  - That said, `createListQueryBoundary` is still best kept separate, because the type argument in that case is the item type itself, rather than the entire data type, and trying to combine that with the generic item causes issues.
