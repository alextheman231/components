This is a new patch release of the `@alextheman/components` package. It includes small, non-breaking changes and should require no refactoring. Please read the description of changes below.
<!-- alex-c-line-release-summary-end -->

## Description of Changes

<!-- user-editable-section-start -->
- Remove the ability to set the background in the Storybook docs.
    - The background in the docs is controlled by Storybook rather than our ModeProvider, which can create some confusion.
    - This changes that so that we just always use the Storybook default background, and then we can change the mode per example.
- Remove the auto-generated code examples from the docs.
    - They tended to be misleading due to the minification leading to incorrect component names.
    - Components that forward a ref often also ended up resulting in `React.ForwardRef` a lot of times in the examples, which over-exposes the internals.
    - As such, for now I have decided to just get rid of them altogether.
    - In the future, however, I would like to have it so that the code examples are generated through the `@example` tag of the JSDoc comment. That way, we get the benefit of being able to see the example in the editor, as well as in the Storybook, and we can be sure that it's the same example both ways.
