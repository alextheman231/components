## Description of Changes

<!-- user-editable-section-start -->
- Add a story for parametric route matching.
    - As explained in the previous release note, this is one of Wouter's biggest strengths over `react-router-dom`.
    - As such, I figured it'd be good to add a story for it to better demonstrate this strength.
    - Essentially, any URL parameters can be parsed with Regex so you can be absolutely sure the value matches the expected format. This is especially helpful for cases where you only want to match IDs of a specific type, rather than all possible ID formats (e.g. filtering by UUID).
